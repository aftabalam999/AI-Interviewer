# Security Policy

## Supported Versions

The following versions of AI-Interviewer receive security updates.

| Version | Supported |
|---------|-----------|
| Latest Release | ✅ |
| Development (main) | ✅ |
| Older Releases | ❌ |

---

## Reporting a Security Vulnerability

If you discover a security vulnerability, please **do not open a public GitHub issue**.

Instead, report it privately by contacting:

**Email:** course.aftabalam@gmail.com

Please include:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Screenshots or proof of concept (if applicable)
- Suggested fix (optional)

---

## Response Timeline

We aim to:

- Acknowledge reports within **48 hours**
- Provide an initial assessment within **7 days**
- Release a fix as quickly as possible depending on severity

---

## Scope

Examples of vulnerabilities that should be reported include:

- Authentication or authorization bypass
- JWT vulnerabilities
- Account takeover
- Sensitive data exposure
- Injection attacks (SQL, NoSQL, Command Injection)
- Cross-Site Scripting (XSS)
- Cross-Site Request Forgery (CSRF)
- Remote Code Execution (RCE)
- Server-side request forgery (SSRF)
- Dependency vulnerabilities
- Misconfigured permissions
- API security issues

---

## Out of Scope

The following are generally considered out of scope:

- Missing security headers without exploitability
- Rate limiting suggestions
- UI or styling issues
- Denial-of-Service testing
- Social engineering attacks
- Issues caused by outdated browsers
- Vulnerabilities in third-party services

---

## Responsible Disclosure

Please allow us reasonable time to investigate and resolve reported vulnerabilities before publicly disclosing them.

We appreciate responsible security research that helps improve AI-Interviewer.
